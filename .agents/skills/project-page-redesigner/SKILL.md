# Project Page Redesigner Skill

## Role

Redesign project pages while keeping the website's existing identity.

## Process

1. Read project page code.
2. Read existing assets.
3. Audit current layout.
4. Identify project story.
5. Select layout direction:
   - editorial
   - cinematic
   - Swiss grid
   - gallery-led
   - motion-led
   - luxury minimal
6. Propose structure.
7. Implement.
8. Review.
9. Update learnings.

## Page structure options

Use combinations of:

- full-screen hero
- editorial title lockup
- metadata strip
- image-led intro
- split text/image section
- full-bleed visual break
- detail grid
- process sequence
- motion module
- final payoff
- next project transition

## Rules

- Do not make all projects identical.
- Do not make layout variation random.
- Use variation based on the project's strongest assets.
- Keep shared brand system consistent.
- Make the strongest visual appear early.
- Avoid long text blocks.
- Avoid image dumps.

## Deliverables

- new layout
- stronger image rhythm
- cleaner hierarchy
- responsive design
- notes in learnings.md
