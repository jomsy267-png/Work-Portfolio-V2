# Asset Rules

## Purpose

This file prevents Codex from damaging brand and project assets.

## Logo rules

- Never recreate logos using image generation unless explicitly asked.
- Never trace or reinterpret the logo.
- Use original logo files from the repo.
- Do not stretch, skew, recolour, or distort logos.
- Preserve clear space around logos.

## Colour rules

- Use existing brand colours.
- If a new shade is needed, derive it from existing tokens and explain why.
- Never introduce random gradients or trendy colours.

## Font rules

- Use existing project fonts.
- Do not replace fonts without permission.
- If a mockup contains typography, preserve the original style as closely as possible.
- Do not allow generated images to invent unreadable fake text.

## Project imagery rules

- Real project imagery is the foundation.
- Image generation may improve context, mockup environment, lighting, framing, or presentation.
- Image generation must not change the project’s actual design work.

## File naming

Use clear names:

project-name_hero_v01.webp
project-name_detail-grid_v01.webp
project-name_motion-loop_v01.mp4
project-name_social-cut_v01.mp4
