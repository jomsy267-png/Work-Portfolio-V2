# Mockup Rules

Date added: 2026-04-26

## Purpose

This file defines the portfolio-wide mockup system for Awwwards-level project pages. It applies to branding, print, signage, web, motion, campaign, production, large-format, and mixed-media projects.

## Core rule

Mockups are evidence, not decoration.

A mockup is useful only when it clarifies medium, scale, material, interaction, or context. If it only makes the page look more designed, remove it.

## Preferred hierarchy

1. Real photography of the actual work.
2. Real installation/context photography.
3. Clean flats from source design files.
4. Carefully composed mockups using preserved real work.
5. AI-assisted environment around preserved real work.

## General mockup rules

- Use one mockup philosophy per project page.
- Preserve actual project work exactly.
- Keep lighting, shadow, perspective, and material believable.
- Let the project type determine the mockup language.
- Use quiet surfaces and neutral contexts that fit the portfolio identity.
- Avoid visual effects that compete with the work.

## Forbidden defaults

Do not use these as generic defaults:

- floating laptops
- rows of phones
- isometric device stacks
- glowing browser frames
- fake glass surfaces
- fake luxury interiors
- PSD-template paper scenes
- unrealistic curled pages
- random shadows/glows
- fake UI or fake copy

## Browser and device frames

Use browser/device frames only when they clarify the medium.

Allowed:

- quiet browser-less plate
- minimal browser chrome
- desktop/mobile pairing when responsive proof matters
- framed interaction capture for real workflow

Avoid:

- multiple chrome styles in one project
- oversized low-resolution screenshots
- live iframe presentation by default
- fake devices that draw more attention than the UI

## Print mockups

Use:

- real object photography
- clean spread flats
- simple tabletop context
- material detail when real

Avoid:

- fake page curl
- fake thick books
- glossy 3D magazine templates
- invented paper texture
- unreadable spread grids

## Signage and environmental mockups

Use:

- real location photography where possible
- human/architectural scale reference
- credible perspective and lighting
- material/detail close-ups

Avoid:

- flat artwork as environmental proof
- fake storefronts with no scale logic
- generated scenes that change the design
- dramatic lighting that hides signage legibility

## AI-assisted mockups

AI assistance is allowed only for context and presentation.

Before generating or editing:

1. Define source asset.
2. Define what must be preserved.
3. Define what may change.
4. Define output role.
5. Define aspect ratio.
6. Define negative prompt.
7. Define approval checkpoint.

Never use AI to invent project work, rewrite text, change logos, change colors, alter typography, or fake results.

## Approval checklist

- Does the mockup preserve the work?
- Does it prove something real?
- Does it fit the page's project type?
- Does it fit the portfolio identity?
- Does it avoid template/mockup stock language?
- Does it remain credible on mobile?
- Is file weight acceptable?

## Merged Synthesis Production Rules

Date added: 2026-04-27

## Hierarchy of acceptability

1. Real photography in context
2. Clean flats from the real design files
3. Environmental composites with credible light and scale logic
4. AI-assisted context only when tightly constrained

Generic PSD-template mockups are not acceptable production assets for this portfolio.

## Real photography standards

- show scale, material, and context honestly
- maintain consistent lighting logic across the page
- include human or architectural reference when scale matters
- use photography that proves the work exists or plausibly exists in its intended medium

## Clean flat standards

- use one quiet background system
- keep perspective square and honest
- preserve typography, color, and edge fidelity
- use flats when photography is unavailable, not as filler

## Environmental composite standards

- use only when context or scale cannot be shown otherwise
- preserve the design exactly
- match light direction, perspective, and surface contact
- keep the background subordinate to the work

## Mockup red flags

- floating laptop or phone renders
- generic desk scenes
- hyper-glossy product rendering for understated work
- multiple mockup philosophies on one page
- fake depth, fake reflections, fake luxury styling
- composites that prove nothing about scale or use

## Lighting / shadow / surface rules

- one dominant light source per project page
- short shadows that imply contact, not levitation
- surface tone should support, not recolor, the work
- visible material truth beats dramatic atmospheric styling

## One mockup philosophy per project page

Choose one dominant presentation language per page:

- real photography page
- clean flat page
- environmental composite page

Mixed-media projects may combine mediums, but the mockup logic still needs one clear governing philosophy.
