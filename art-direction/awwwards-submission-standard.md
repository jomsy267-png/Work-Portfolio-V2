# Awwwards Submission Standard

## Purpose

This website is being prepared for Awwwards submission.

The goal is not to chase trends. The goal is to create a memorable, polished, high-craft portfolio experience that can be judged seriously by designers, creative developers, and art directors.

## Core judging areas

Every page must be reviewed through:

1. Design
2. Usability
3. Creativity
4. Content
5. Animation / transitions
6. Responsive design
7. Accessibility
8. Performance / WPO
9. Semantics / SEO
10. Markup / metadata

## Design standard

The design must feel:

- intentional
- distinctive
- premium
- portfolio-worthy
- art-directed
- brand-consistent
- not templated
- not generic
- not AI-looking

Strong design means:
- clear hierarchy
- excellent spacing
- confident typography
- strong image sequence
- memorable hero
- polished details
- consistent visual system

## Usability standard

The site must still be easy to use.

Check:
- navigation clarity
- scroll clarity
- readable text
- understandable project flow
- clear project context
- obvious next actions
- no confusing interactions
- no motion that blocks browsing

## Creativity standard

The site needs a point of view.

Creativity can come from:
- unique project page rhythm
- custom transitions
- strong case-study storytelling
- unexpected but useful layout
- cinematic image sequencing
- interactive details
- motion that reveals the work
- personality rooted in the designer's identity

Avoid:
- random gimmicks
- trendy effects with no meaning
- copied Awwwards layouts
- overused bento grids
- fake 3D/glass styling
- interaction for interaction's sake

## Content standard

Project pages must explain enough without becoming long case-study essays.

Each project should communicate:
- what the project is
- why it matters
- what role was played
- what the visual idea is
- what the craft/result is
- why the work is worth seeing

Use short, confident writing.

## Motion standard

Motion must feel custom and controlled.

Good motion:
- improves storytelling
- reveals hierarchy
- creates atmosphere
- supports the brand
- feels smooth
- respects performance

Bad motion:
- delays the user
- feels generic
- repeats everywhere
- hides weak layout
- creates scroll jank
- hurts mobile

## Performance standard

Awwwards-level does not mean heavy.

Check:
- optimized images
- compressed video
- lazy loading
- no unnecessary animation libraries
- no layout shift
- smooth scrolling
- mobile performance
- reduced-motion fallback

## Accessibility standard

Creative websites still need accessibility.

Check:
- readable contrast
- semantic headings
- alt text
- keyboard navigation
- focus states
- reduced motion support
- no text trapped inside images when avoidable

## Final scoring

Before calling a page complete, score:

| Category | Target |
|---|---:|
| Design | 8.5+ |
| Usability | 8+ |
| Creativity | 8.5+ |
| Content | 8+ |
| Motion | 8+ |
| Mobile | 8+ |
| Performance | 8+ |
| Accessibility | 8+ |

Anything below target must be refined.
