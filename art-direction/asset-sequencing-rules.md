# Asset Sequencing Rules

Date added: 2026-04-26

## Purpose

This file defines how project assets should be ordered across portfolio project pages. The goal is Awwwards-level rhythm: authored, image-led, clear, and specific to each project type.

## Core rule

Sequence by story role, not by file order.

Every asset should do one job:

- establish
- deepen
- prove
- detail
- transition
- close

## Base rhythm

Use wide / medium / detail rhythm:

1. Wide: establish atmosphere, scale, or system.
2. Medium: show application or relationship.
3. Detail: show craft, typography, material, UI state, or production quality.
4. Quiet: give space or concise context.
5. Impact: return with a strong visual.
6. Payoff: close with the strongest remaining image or composition.

## Quiet / impact rhythm

Do not make every section loud.

Alternate:

- full-bleed / contained
- overview / detail
- image / copy
- static / motion
- dense / spacious

The page should build. It should not feel like a gallery where each asset has equal weight.

## Asset count guidance

- Simple project: 6-10 visual beats.
- Medium project: 10-16 visual beats.
- Complex mixed-media project: 16-24 visual beats only if chapters are clear.

Omit assets when they repeat a point already made better by another asset.

## Full-bleed rules

Full-bleed is reserved for:

- strong hero-quality imagery
- environmental scale
- campaign key visuals
- high-resolution final work
- major chapter transition
- final payoff

Do not use full-bleed for:

- low-resolution screenshots
- weak process images
- utility UI screens
- repetitive brand boards

## Pairing rules

Pair assets by relationship:

- overview + detail
- before + after
- flat + context
- desktop + mobile
- system + application
- process + result

Do not pair assets only because they are the same size.

## Screenshot sequencing

Website screenshots should follow:

1. strongest screen or interface moment
2. workflow or page-family proof
3. detail/support screen
4. responsive proof if strong
5. final digital-system plate

Avoid:

- raw long screenshot dumps
- every page shown equally
- mobile screenshots as filler
- utility pages as final payoff

## Print sequencing

Print/editorial projects should follow:

1. object or cover
2. strongest spread
3. spread rhythm
4. type/detail/material
5. final physical presence

Avoid showing every spread.

## Campaign sequencing

Campaign pages should follow:

1. campaign idea/key visual
2. primary execution
3. channel rollout
4. detail or social/motion proof
5. integrated campaign payoff

Avoid channel dumps.

## Signage / environmental sequencing

Environmental pages should follow:

1. space/scale context
2. primary sign or installation
3. system/detail
4. human or architectural scale proof
5. final environmental payoff

Avoid leading with flat artwork.

## Motion sequencing

Motion pages should follow:

1. poster frame or hero loop
2. key sequence
3. still frame grid
4. motion logic/process if useful
5. final video/reel

Avoid multiple autoplay loops in the same viewport.

## Mixed-media sequencing

Mixed-media pages need one throughline:

1. anchor image
2. thesis
3. primary medium chapter
4. supporting medium chapter
5. bridge/detail moment
6. integrated final payoff

Do not give each medium equal weight by default.

## Ending rules

The final image before next-project should feel intentional.

Good endings:

- integrated campaign wall
- final artifact in context
- strongest system overview
- calm closing image
- final motion poster

Weak endings:

- leftover screenshot
- utility section
- repeated process image
- small grid with no hierarchy

## Merged Synthesis Sequencing Standard

Date added: 2026-04-27

## Wide / medium / detail rhythm

- Sequence assets editorially:
  wide first, medium second, detail third.
- Repeat the rhythm only when the page earns another cycle.
- Do not open with detail unless the thesis itself is microscopic or materially driven.

## Full-bleed vs contained rhythm

- Full-bleed signals authority or reset.
- Contained layouts signal reading, comparison, or close inspection.
- Alternate impact and quiet; do not stack full-bleed images until they cancel each other out.

## Pairing by relationship, not symmetry

- Pair images because they explain one another.
- Do not pair only because two rectangles happen to fit a row.
- Desktop/mobile, before/after, wide/detail, system/application are legitimate pairings.

## Aspect-ratio discipline

- Choose two or three working aspect-ratio families per page.
- Hold them consistently.
- Random ratio shifts read as an asset dump rather than art direction.

## Image-count guidance

- Most project pages should stay in the rough range of 12 to 18 meaningful assets.
- If a page needs more, chaptering and editing quality must get stricter, not looser.
- Asset abundance is not proof of quality.

## Hero chosen last

- The hero should be chosen after the page story is sequenced.
- The strongest single asset in isolation is not always the best hero.
- Choose the asset that previews the thesis and the page's internal rhythm.

## Weak asset removal rules

Remove or de-emphasize assets that:

- repeat information already proven
- are lower quality than the surrounding set
- force awkward scale or layout compromises
- exist only for completeness
- flatten the page's rhythm

## Strong ending rules

- End on an integrated payoff, calm closing image, or final system authority moment.
- The last project asset must prepare the next-project handoff.
- Never end on utility, leftovers, or a weak gallery fragment.
