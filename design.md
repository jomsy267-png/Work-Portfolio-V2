# design.md — Portfolio Design System

## Purpose

This file defines the design expectations for the portfolio website.

The website already has a brand identity. The job is not to redesign the brand. The job is to improve how projects are displayed within the brand.

## Design personality

The website should feel:

- premium
- calm
- editorial
- confident
- refined
- cinematic when useful
- design-led
- visually restrained
- technically sharp
- personal but not casual

## Core design principles

### 1. Restraint first

High-end design is usually not more decoration. It is better hierarchy, better spacing, better rhythm, and better decisions.

### 2. One dominant idea per page

Every project page should have one main visual story. Do not mix too many concepts.

### 3. Strong hero moment

The first viewport should immediately communicate:
- what the project is
- why it matters visually
- the level of craft
- the mood of the work

### 4. Editorial pacing

Project pages should feel like a magazine feature or studio case study, not a dump of images.

Use:
- quiet intro
- strong image moment
- detail rhythm
- full-bleed break
- process or system section
- final payoff

### 5. Negative space is part of the design

Do not fill every gap. Use space to create confidence.

### 6. Typography must be intentional

Typography should feel:
- clear
- confident
- structured
- spacious
- not over-styled

### 7. Motion must have purpose

Motion should support:
- reveal
- hierarchy
- transition
- storytelling
- atmosphere

Motion should not exist just because it can.

## Project page tone

Each project page should feel custom, but all pages should belong to the same portfolio.

Variation is allowed in:
- grid structure
- image scale
- scroll rhythm
- hero treatment
- motion moment
- case-study sequence

Variation is not allowed in:
- brand identity
- core fonts
- core colours
- logo usage
- overall site personality
